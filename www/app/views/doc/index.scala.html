api.json

Fully describe your REST service in an intuitive json format - start by creating a service below or downloading an example.

Very intuitive format that is simple to pick up - either from online docs or from user contributed public services.

Convention over configuration

Start by describing your models, and then map your models to resources exposing the specific operations you would like to make available.

api.json is designed to guide developers to think carefully about their APIs, making it simple to create services that align to the REST standard. By favoring convention over configuration in a few key places - developers are more likely to create great REST APIs.

Simple native client libraries with no dependencies

Download native client libraries with no (or almost no) dependencies. We take special care to make sure the client libraries are a pleasure to use.

Example from ruby:

      client = MyService::Client.new("http://localhost:8000")

      organizations = client.organizations.get(:limit => 10, :offset => 0)
      organizations.each do |org|
        puts "Org %s is named %s" % [org.id, org.name]
      end

      neworg = client.organizations.post(:name => "My org")
      puts "Created new org named %s" % neworg.name

Example from scala:

      val client = com.gilt.apidoc.Client.new("http://localhost:8000")

      val organizations = client.organizations.get(limit = Some(10), offset = Some(0))
      organizations.foreach { org =>
        println(s"Org ${org.name} is named ${org.id}")
      }

      val neworg = client.organizations.post(name = "My org")
      println(s"Created new org named ${neworg.name}")
